Moonchild AI とは

Moonchild AI は「Chat for Product Design」を掲げる AI ネイティブのデザインプラットフォーム。公式キャッチコピーは次の通り。

Stop designing from scratch. Your design system already has the pieces. Moonchild assembles them. Paste a PRD, get production-ready screens. Export to code when it's right.

最大の特徴：システムファースト生成

Uizard や初期 Figma Make のような従来ツールは「ランダム UI を生成 → 後でトークンを手動置換」という流れが必要だった。Moonchild は逆で、最初からユーザーの既存 DS を制約として与える。

  • Figma ファイル / GitHub リポジトリ / Live URL から DS をインポート
  • トークン / コンポーネント / スペーシング / ブレイクポイント / タイポを正確に抽出
  • すべての生成画面が ユーザーの DS に 100% 準拠 (semantic naming で抽出精度 95%)
  • 結果として「あなたの UI」が生成 → リワークが激減

主な機能

  • PRD / テキストから 完全な DS + 全画面 + インタラクティブプロトタイプ を生成
  • 3 バリエーション以上の同時出力 (ダッシュボード / オンボーディング 等)
  • Gold / Silver モード — 精度と速度の切替
  • Figma 双方向連携 (インポート / エクスポート)
  • MCP (Model Context Protocol) 対応 — スクショではなく構造化データをそのまま AI コーディングエージェントに渡す
💡
公式実績 (2026年5月時点)
550,000 件以上のプロトタイプ生成 · 67,000 件以上のデザインエクスポート。

OpenAI Codex (2026 最新版)

OpenAI Codex は 2025〜2026 年に大きく進化した クラウドネイティブの AI コーディングエージェント。2021 年の旧コード補完モデルとは別物。

  • ユーザーのリポジトリをプリロードした 隔離クラウド sandbox でタスクを実行
  • コードの作成・テスト・デバッグ・PR レビュー・ブラウザ操作まで自律処理
  • GPT-5.5-Codex モデル (大規模コンテキスト)
  • デスクトップアプリ + CLI + VS Code 拡張
  • MCP 対応 により Moonchild から構造化データを直接受信
🔗
MCP がもたらす本質的な変化
スクリーンショットを「見て」コードを書くのではなく、正確なトークン / コンポーネント情報を「受け取って」生成するため、デザインとコードのドリフトが理論上ゼロ。

セットアップ手順

Moonchild の始め方

  1. moonchild.ai にサインアップ
  2. waitlist の場合は hello@moonchild.ai に使用目的を添えて連絡
  3. ログイン → プロジェクト作成
  4. Figma の DS をインポート (最重要ステップ)
  5. PRD を貼って画面生成開始

Codex + MCP 接続

  1. ChatGPT アカウントで Codex デスクトップアプリをダウンロード (or CLI インストール)
  2. ターミナルで以下を実行
bash setup.sh 
# Codex CLI のインストールと認証
npm install -g @openai/codex
codex auth login
  1. Moonchild で MCP URL を発行し Codex に追加
bash mcp-connect.sh 
# Moonchild の MCP エンドポイントを Codex に登録
codex mcp add moonchild --url [発行された URL]
  1. 接続確認 → 通常のプロンプトでコード生成開始

初回は 30〜60 分かかるが、2 回目以降は 15 分以内 で完了するようになる。

限界と現実的な期待値

⚠
「Production-Ready」の正しい解釈
DS に完全準拠した「初回パス」を意味する。クライアントフィードバック / エッジケース調整 / 最終 a11y 確認は依然として人間の仕事。
  • 強力な既存 DS があるほど効果が高い — DS が未熟だと生成品質も低下する
  • アクセシビリティ / パフォーマンスの最終確認は人間 — 自動生成された WCAG 準拠は鵜呑みにせず、Lighthouse / axe で実測
  • 戦略・差別化要素は人間が決める — ツールは「実装の高速化」、創造性そのものは代替しない

このワークフローは「人間の仕事を奪う」のではなく、「人間がより戦略的・創造的な仕事に集中できる」ようにする強力なツールとして捉えるのが妥当。
